# LLM 프롬프트 엔지니어링 가이드

Large Language Model(LLM)의 성능을 최대로 끌어내기 위해서는 좋은 프롬프트가 필수입니다.

## 프롬프트 엔지니어링이란?

AI 모델에게 원하는 결과를 얻기 위해 입력(프롬프트)을 최적화하는 기술입니다.

## 기본 원칙

### 1. 명확하고 구체적으로

**나쁜 예:**
```
코드 작성해줘
```

**좋은 예:**
```
Python으로 1부터 100까지의 소수를 찾아 리스트로 반환하는
함수를 작성해주세요. 함수명은 find_primes로 해주세요.
```

### 2. 맥락 제공

**나쁜 예:**
```
이 코드 수정해줘
```

**좋은 예:**
```
아래 Python 코드가 있습니다. 이 코드는 사용자 로그인을
처리하는 함수입니다. 현재 비밀번호를 평문으로 저장하는
문제가 있어서, bcrypt를 사용해 해시로 저장하도록 수정해주세요.

[코드]
```

### 3. 출력 형식 지정

```
다음 데이터를 분석하고 결과를 아래 형식으로 제공해주세요:

## 요약
[한 문장 요약]

## 주요 발견
- 발견 1
- 발견 2

## 권장 사항
1. 첫 번째 권장
2. 두 번째 권장
```

## 고급 기법

### 1. Few-shot Learning

예시를 제공하여 패턴을 학습시킵니다.

```
다음 형식으로 감정을 분류해주세요.

예시:
입력: "오늘 정말 행복해!"
출력: 긍정

입력: "너무 화가 나서 미치겠어"
출력: 부정

입력: "그냥 그래"
출력: 중립

이제 분류해주세요:
입력: "이 영화 정말 재밌었어!"
출력:
```

### 2. Chain-of-Thought (CoT)

단계별 사고 과정을 유도합니다.

```
다음 문제를 단계별로 풀어주세요.

문제: 철수는 사과 5개를 가지고 있습니다. 영희에게 2개를 주고,
어머니에게 1개를 받았습니다. 철수가 가진 사과는 몇 개인가요?

단계별로 생각해봅시다:
1. 처음 철수가 가진 사과: ?
2. 영희에게 준 후: ?
3. 어머니에게 받은 후: ?

최종 답:
```

### 3. Role Playing

역할을 부여하여 전문성을 높입니다.

```
당신은 10년 경력의 시니어 백엔드 개발자입니다.
주니어 개발자가 작성한 아래 코드를 리뷰해주세요.
보안, 성능, 가독성 측면에서 피드백을 제공하고,
개선된 코드 예시를 보여주세요.

[코드]
```

### 4. Self-Consistency

여러 번 질문하여 일관된 답변 확보

```
이 문제에 대해 3가지 다른 접근 방식으로 생각해보고,
가장 합리적인 답을 선택해주세요.
```

## 프롬프트 템플릿

### 코드 리뷰
```
언어: [프로그래밍 언어]
목적: [코드의 목적]
리뷰 관점: [보안/성능/가독성/버그]

[코드]

위 코드를 리뷰하고 개선점을 제안해주세요.
```

### 번역
```
아래 텍스트를 [목표 언어]로 번역해주세요.
전문 용어는 유지하고, 자연스러운 표현으로 번역해주세요.

[텍스트]
```

### 요약
```
아래 글을 [목적]을 위해 [길이]로 요약해주세요.
핵심 키워드를 포함하고, 불릿 포인트로 정리해주세요.

[글]
```

## 피해야 할 실수

1. **너무 모호한 지시**
2. **상충되는 요구사항**
3. **너무 긴 프롬프트** (핵심만 간결하게)
4. **편향된 유도 질문**

## 마무리

좋은 프롬프트 = 좋은 결과

프롬프트 엔지니어링은 연습이 필요합니다. 다양한 실험을 통해 자신만의 노하우를 쌓아가세요.
